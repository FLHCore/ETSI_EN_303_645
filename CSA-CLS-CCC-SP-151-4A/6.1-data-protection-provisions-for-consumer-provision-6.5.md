## 6.1 – 消費者資料保護條款 (DATA PROTECTION PROVISIONS FOR CONSUMER)

### 條款 6.5 (Provision 6.5)

若從消費型物聯網設備與服務收集遙測數據 (telemetry data)，則必須向消費者提供關於收集了哪些遙測數據、如何使用、由誰使用以及使用目的之資訊。

#### 最低要求 (Minimum Requirements)

* 若設備收集遙測數據，必須提供資訊以告知使用者**收集了哪些遙測數據**、**如何使用**、**由誰使用**以及**使用目的**。
* 遙測數據的範例包括但不限於：
    * 認證與授權 (Authentication and authorisation)
    * 系統程序 (System processes)
    * 系統檔案變更 (System file changes)
    * 使用者行為 (User behaviour)

#### 佐證證據 (Supporting Evidence)

* 開發商應宣告設備、其配套應用程式及相關服務（例如：行動應用程式、GUI、週邊設備等）是否收集遙測數據。
* 若設備收集遙測數據，開發商應解釋**收集了什麼**、**數據如何被使用**、**由誰使用**以及**為了什麼目的**。
* 開發商亦應提供證據（例如：截圖、影片、文件等），顯示這些資訊是如何提供給使用者的。

#### 評估 (Assessment)

* 若設備、其配套應用程式及相關服務收集遙測數據，評估人員將檢查是否已提供資訊給使用者，說明收集了哪些遙測數據、如何使用、由誰使用以及為了什麼目的。

### 開發者深度導讀 (針對 Gateway / App / Cloud 開發商)

#### 本章節核心要求與意涵

本條款針對的是開發者常用來改善產品的**「埋點 (Analytics)」**與**「除錯日誌 (Debug Logs)」**。

1.  **重新定義「遙測 (Telemetry)」**：
    * 許多開發者認為「我只收集 Crash Report，沒有收集使用者的名字，所以不算個資」。
    * **合規修正**：在 CLS(IoT) 規範下，系統運作狀態（如 CPU Loading, Memory Usage, Crash Stack Trace, Uptime）都屬於「遙測數據」。即便它不是 PII，也必須履行**告知義務**。
2.  **使用者行為分析 (User Behavior)**：
    * 如果您有統計「使用者每天開燈幾次」或「使用者最常使用哪個功能」，這屬於 User Behaviour Telemetry。這類數據通常用於優化 UX，但必須明確告知使用者。
