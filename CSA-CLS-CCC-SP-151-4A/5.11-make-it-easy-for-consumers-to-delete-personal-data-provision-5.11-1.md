## 5.11 - 讓消費者容易刪除個人數據 (MAKE IT EASY FOR CONSUMERS TO DELETE PERSONAL DATA)

### 條款 5.11-1 (Provision 5.11-1)

使用者必須被提供一種功能，使其能以簡單的方式從設備中刪除使用者數據。

#### 最低要求 (Minimum Requirements)

* 設備必須具備**至少一項功能**（例如：透過 GUI、配套行動應用程式、硬體重置按鈕等），允許使用者刪除儲存在其上的**個人數據**、**使用者配置設定**與**密碼學材料**（例如：使用者密碼、金鑰等）。
* 必須**告知**使用者關於這些功能的資訊（例如：設備上硬體重置按鈕的用途等），以解釋使用者如何執行刪除設備上儲存數據所需的動作。

#### 佐證證據 (Supporting Evidence)

* 開發商應列出所有能刪除儲存在設備上使用者數據的功能。
* 開發商應提供證據（例如：使用者指引文件、顯示如何刪除設備上個人數據與硬體重置按鈕用途的網頁截圖等），證明該功能**存在**且**已讓使用者知悉**。

#### 評估 (Assessment)

* 評估人員將檢查設備是否具備至少一項能從設備中刪除使用者數據的功能。
* 評估人員將檢查證據，以判斷這些功能是否已讓使用者知悉。

### 開發者深度導讀 (針對 Gateway / App / Cloud 開發商)

#### 本章節核心要求與意涵

本條款體現了**「被遺忘權 (Right to be Forgotten)」**在硬體設備上的實作，也就是俗稱的**「恢復原廠設定 (Factory Reset)」**。

1.  **刪除範圍的完整性**：
    * 許多開發者實作的 Reset 只是「清除設定 (Clear Config)」，卻忘了清除：
        * **Log 檔**（可能包含 IP 或使用習慣）。
        * **Wi-Fi Credentials**（如果是儲存在獨立的 Supplicant 檔）。
        * **Pairing Keys**（與雲端或 App 配對的 Token）。
    * **合規紅線**：執行刪除功能後，設備必須回到「剛出廠的空白狀態 (Clean Slate)」。任何殘留的 PII (Personal Identifiable Information) 都會導致測試失敗。
2.  **操作的可及性 (Accessibility)**：
    * 必須有「至少一種」方法。通常建議同時提供**軟體重置**（GUI）與**硬體重置**（實體按鈕），以防設備當機或無法連線時無法刪除資料。
3.  **告知義務**：
    * 僅僅做了 Reset 按鈕是不夠的，說明書必須明確寫出「長按 10 秒可清除所有資料」。

#### 具體落地實現方案建議

* **系統設計與資料清理腳本**：
    * **分區格式化 (Partition Formatting)**：
        * 最乾淨的做法是直接格式化存放使用者資料的 Flash Partition (如 `/data` 或 `/overlay`)。
        * 指令範例：`mkfs.jffs2 --root=/overlay` 或 `rm -rf /data/*`。
    * **安全元件 (SE) 重置**：
        * 如果金鑰存放在 Secure Element，務必呼叫 SE 的 Reset API，清除內部的 Key Slot。
    * **雲端解除綁定 (Unbinding)**：
        * 雖然條款主要針對「設備上的數據」，但最佳實務是 Reset 時同時發送 API 通知雲端「解除綁定」，避免設備重置後，雲端還保留著舊的關聯。

* **證據收集策略**：
    * **硬體按鈕照片**：
        * 拍攝機身背後的 "Reset" 孔位。
    * **GUI 截圖**：
        * 截取管理介面中的「恢復原廠設定」按鈕。
    * **手冊掃描**：
        * 掃描說明書中關於 "Resetting the Device" 的章節，證明有教導使用者如何操作。

* **測試與驗證**：
    * **殘留數據檢查 (Data Remanence Test)**：
        * **步驟 1**：正常使用設備，設定 Wi-Fi，產生一些 Log，登入帳號。
        * **步驟 2**：執行 Factory Reset。
        * **步驟 3**：透過 Serial Console 或 JTAG 進入系統，檢查 `/etc/config/`, `/var/log/`, `/data/` 等目錄。
        * **預期結果**：所有使用者產生的檔案都應消失或回到預設值。
